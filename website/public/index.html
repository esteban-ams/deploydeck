Features

Everything you need,
nothing you don't

A single binary that handles webhooks, deployments, health checks, and rollbacks. No agents, no complex setup.

Webhook-Driven

Native support for GitHub and GitLab webhooks. Trigger deploys from CI/CD pipelines or repository push events with HMAC authentication.

Pull or Build

Pull pre-built images from registries or clone your repo and build directly on the server. Choose the mode that fits your workflow.

Health & Rollback

Automatic health verification after deploy. If something breaks, DeployDeck rolls back to the previous working image instantly.

Auth Built In

Three auth methods out of the box: GitHub HMAC signatures, GitLab tokens, and simple shared secrets. Secure by default.

Single Binary

Written in Go, compiles to a single binary with zero external dependencies. Drop it on your server and you are ready to go.

Compose Native

Works directly with Docker Compose. No custom orchestration layer, no lock-in. Your existing compose files work as-is.

How It Works

Three steps to automated deployments

From configuration to continuous deployment in under five minutes.

1

Configure Your Services

Define your services in a simple YAML config. Set the deploy mode, compose file path, health check endpoints, and rollback options.

2

Set Up Your Webhook

Point your GitHub Actions, GitLab CI, or repository webhook to your DeployDeck endpoint. Add your shared secret for authentication.

3

Push and Deploy

Push your code or trigger your CI pipeline. DeployDeck receives the webhook, pulls or builds your image, and deploys with health verification.

Quick Start

Up and running in minutes

Install DeployDeck, add your config, and start deploying.

terminal
# Install DeployDeck $ curl -sL https://github.com/esteban-ams/deploydeck/releases/latest/download/deploydeck_linux_amd64 -o /usr/local/bin/deploydeck $ chmod +x /usr/local/bin/deploydeck # Or pull the Docker image $ docker pull ghcr.io/esteban-ams/deploydeck:latest # Create your config $ cat config.yaml port: 8000 webhook_secret: "your-secret-here" services: api: mode: pull compose_file: "./docker-compose.yml" health_check: url: "http://localhost:3000/health" timeout: 30s rollback: enabled: true # Start DeployDeck $ deploydeck --config config.yaml [INFO] DeployDeck listening on :8000
